# 繃布報價系統使用說明

適用對象：馬鈴薯沙發內部同事
目前版本：v0.3.2（2026-03-22）

## 1. 這份文件放哪裡

本文件存放於：

- `docs/使用說明.md`

系統內建「使用說明」頁面，入口在側邊欄 `使用說明`，網址為 `/help`。

## 2. 系統怎麼開

### 本機啟動

在專案根目錄執行：

```bash
./RESTART_QUOTE_SYSTEM.sh
```

系統會在下列網址啟動：

- `http://localhost:3001`

也可以直接執行：

```bash
npm run dev -- -p 3001
```

### 環境需求

系統正式資料會寫入 Google Sheets，啟動前要有這兩個環境變數：

- `GOOGLE_SHEETS_SPREADSHEET_ID`
- `GOOGLE_SERVICE_ACCOUNT_KEY`

### 第一次建置

如果是第一次接上新的 Google Sheets 試算表，需要由管理者先呼叫一次：

- `POST /api/sheets/init`

用途是建立系統需要的工作表與預設欄位。

### 登入方式

目前系統沒有登入機制，知道網址即可使用。

## 3. 新人上手 3 分鐘流程

如果你今天第一次用，先照這個流程跑一次，不要一開始就看全部功能。

### 第 1 分鐘：先確認基礎資料

1. 打開左側 `系統設定`。
2. 確認公司資訊、營業稅、通路倍率大致正確。
3. 如果今天只是開單，不要先改倍率，先照現有設定使用。

### 第 2 分鐘：確認客戶與案件

1. 打開 `客戶管理` 搜尋客戶名稱或電話。
2. 如果查不到，再按 `新增客戶`。
3. `公司名稱` 是必填，其餘欄位可以先填常用資料。
4. 如果這次只是單次報價，也可以直接回報價工作台手動輸入，不一定要先建完整客戶。

### 第 3 分鐘：建立第一張報價

1. 回到 `報價工作台`。
2. 在 `客戶資訊` 選客戶，或選 `＋ 新客戶（手動輸入）`。
3. 填 `案場名稱`。
4. 按 `用計算器算` 建第一個品項；如果是簡單項目，也可以直接按 `新增品項` 手填。
5. 檢查右下角 `總額`。
6. 按 `預覽 PDF` 看報價單外觀。
7. 按 `儲存` 完成第一版。

### 儲存後去哪裡找

- 同一案件後續追版本：去 `案件管理`
- 要查單號、改狀態、看舊單：去 `報價紀錄`

### 新人先記這 4 件事

- 正式資料存在 Google Sheets，不是存在你這台電腦。
- 草稿只存在目前瀏覽器，換電腦不會跟著過去。
- 同一案件如果要改價，優先用 `複製` 建立新版本，不要直接覆蓋舊版。
- 如果只想先把單做出來，先求能儲存、能出 PDF，再回頭補資料。

## 4. 建議日常使用流程

建議依這個順序操作：

1. 先到「系統設定」確認倍率、稅率、公司資料。
2. 到「客戶管理」建立或更新客戶資料。
3. 到「材質資料庫」建立常用布料/皮革資料。
4. 在「報價工作台」建立報價內容、預覽 PDF、儲存版本。
5. 到「案件管理」追蹤同一案件的不同版本。
6. 到「報價紀錄」查舊單、改狀態、看歷史快照。

## 5. 各頁面怎麼用

### 5.1 報價工作台

![報價工作台](/screenshots/02-quote-editor.png)

這是主要工作區，頁籤分成：

- `報價編輯`
- `營運統計`

日常操作幾乎都在 `報價編輯`。

#### 上方按鈕

![上方按鈕列與 PDF 預覽](/screenshots/07-pdf-preview.png)

- `新建報價`：清空目前內容，重新開始一張新單。
- `儲存 / 更新`：將目前內容寫入系統。
- `複製`：可建立同案新版本，或套用成新案件。
- `複製文字版`：把報價摘要複製到剪貼簿，方便貼到 LINE。
- `預覽 PDF`：產生報價單 PDF 預覽。

#### 客戶資訊

- 可直接從 `選擇客戶` 下拉選既有客戶。
- 也可選 `＋ 新客戶（手動輸入）`，直接手填本次客戶資料。
- 常用欄位包含公司名稱、聯絡人、電話、統編、案場名稱、Email。

#### 品項編輯區

- `新增品項`：手動新增一列。
- `用計算器算`：開啟繃布計算器，自動帶入作法、尺寸、材質、單價。
- `常用範本`：插入單一常用品項。
- `套用整單範本`：整批替換目前所有品項。
- 可拖曳排序品項順序。
- 可用 `Undo / Redo` 或快捷鍵 `Cmd/Ctrl+Z`、`Cmd/Ctrl+Shift+Z`。

每一列可填：

- 商品名稱
- 規格
- 數量
- 單位
- 單價
- 金額
- 備註
- 附圖

#### 補充說明與條款

- `補充說明` 會顯示在報價單上。
- 可附上 1 張示意圖。
- `備註 / 條款` 會作為報價條款內容。

#### 稅額與總額

- 可勾選是否計入營業稅。
- 系統會即時計算小計、稅額、總額。

#### 儲存邏輯

- 新報價如果還沒有案件，系統會自動建立案件。
- 新版報價會存成「案件 + 報價 + 報價版本 + 報價版本明細」。
- 如果是已存在的版本，按 `更新` 會直接覆寫該版本內容。
- `複製` 可做兩件事：
  - `建立新版本（議價調整）`
  - `套用為新報價（新案子）`

### 5.2 客戶管理

![客戶管理](/screenshots/08-clients.png)

用途：建立與維護客戶主檔。

可做的事：

- `新增客戶`
- 搜尋公司、聯絡人、電話、統編
- 編輯客戶資料
- 查看該客戶的報價歷史

重要欄位：

- `公司名稱` 為必填
- 可設定 `預設通路`
- 可設定 `佣金模式`

### 5.3 案件管理

![案件管理](/screenshots/10-cases.png)

用途：以案件角度查看報價進度與版本。

可做的事：

- `新增案件`
- 搜尋案件名稱、客戶、案件編號
- 展開案件查看底下所有報價方案與版本
- 直接開啟某一版本到報價工作台編輯
- 複製某一版本成新版本或新案件

適合用在：

- 同一客戶同一案場有多次調價
- 需要追蹤最近送出與下次追蹤日期

### 5.4 材質資料庫

![材質資料庫](/screenshots/09-materials.png)

用途：管理布料、皮革等材質主檔。

可做的事：

- `新增材質`
- 搜尋品牌、系列、色號、色名
- 編輯材質
- 停用材質

重要欄位：

- `品牌`、`系列` 為必填
- 可填色號、色名、供應商、門幅、進價、牌價、交期、庫存狀態
- 支援圖片上傳（透過 Cloudinary）
- 支援收藏與最近使用標記

### 5.5 報價紀錄

用途：查詢既有報價紀錄與舊版資料。

可做的事：

- 依客戶、案場、單號、日期、狀態搜尋
- 調整報價狀態
- 編輯舊報價
- 複製舊報價
- 查看版本歷史快照
- 刪除報價

注意：

- 這個頁面主要偏向舊版 `報價紀錄 / 報價明細` 資料流。
- 新版日常追蹤請優先看 `案件管理`。

### 5.6 佣金管理

用途：追蹤與管理設計師/合作夥伴的佣金結算。

可做的事：

- 查看待結算佣金
- 依合作夥伴、時間範圍篩選
- 標記佣金為已付款
- 匯出佣金報表

支援的佣金模式：

- **賺價差**：設計師以設計師價進貨，自行加價報給屋主
- **返佣**：以屋主價開立，系統自動計算返佣金額
- **固定金額**：每單固定佣金，依全單品項自動分攤
- **無佣金**：批發通路或終端屋主

### 5.7 系統設定

![系統設定](/screenshots/11-settings.png)

用途：控制整套系統的報價基準。

可調整內容：

- 通路倍率（批發/設計師/屋主，精度 0.05）
- 品質溢價（%）
- 損耗率（%）
- 布料折數（%）
- 營業稅（%）
- 佣金模式與返佣比例
- 報價有效天數
- 公司資訊（名稱、地址、電話、LINE ID）

注意：

- 這裡的公司資訊會顯示在 PDF 報價單。
- 修改後要按 `儲存設定` 才會寫回 Google Sheets。

## 6. 繃布計算器怎麼用（含 v0.3.1 & v0.3.2 新功能）

**基本模式（每片 × 數量）**：

![繃布計算器 - 基本模式](/screenshots/03-calculator-modal-basic.png)

**進階模式（整面 ÷ 分片）**：

![繃布計算器 - 進階模式](/screenshots/03-calculator-modal-advanced.png)

在報價工作台按 `用計算器算` 後，會開啟智慧計算器。計算器會根據你選擇的作法與輸入的尺寸，自動計算成本與建議售價。

### 基本流程

1. **選擇通路**：批發 / 設計師 / 屋主
2. **選擇作法**：平貼 / 單面板 / 活動板 / 單面榻 / 雙面榻 / 泡棉內裡
3. **輸入尺寸與數量**（見下方「輸入模式」說明）
4. **選擇厚度**（如適用）
5. **選擇材質**：從材質資料庫選取，或使用自訂成本
6. **勾選額外加工**：皮革加工、加車裡布、背車止滑布、代釘電源孔
7. **設定施工條件**（見下方「施工加給」說明）
8. **確認單價後插入品項**

### ✨ NEW: 輸入模式切換（v0.3.2）

計算器支援兩種輸入模式：

#### 模式 1：每片 × 數量（預設）

適用情境：每片尺寸相同，數量多片

- 輸入：單片寬度、單片高度、數量
- 範例：床頭板 180×120cm，做 2 片

#### 模式 2：整面 ÷ 分片（v0.3.2 新增）

適用情境：整面牆分割成多片板材

- 輸入：整面寬度、整面高度
- 選擇分片方向：
  - **橫切（沿高度）**：高度平分，寬度不變
  - **直切（沿寬度）**：寬度平分，高度不變
- 調整分片數量（± 按鈕）
- 系統自動計算：每片尺寸

範例：整面牆 360×240cm，橫切 3 片 → 每片 360×80cm

### ✨ NEW: 才數進位模式對比（v0.3.2）

在「整面 ÷ 分片」模式下，系統會同時顯示兩種才數計算方式：

#### 逐片進位（預設）

- 每片獨立計算才數並進位
- 再乘以數量得總才數
- 範例：每片 6.7 才 → 進位 7 才 × 3 片 = 21 才

#### 整面進位

- 所有片加總後才進位一次
- 通常比逐片進位節省才數
- 範例：6.7 才 × 3 片 = 20.1 才 → 進位 21 才

**橘色差異提示**：

- 當兩種模式才數不同時，會顯示橘色提示框
- 顯示詳細計算過程與成本差異
- 幫助你選擇最適合的計價方式

### ✨ NEW: 施工加給分級制（v0.3.1）

系統會根據**安裝高度**與**板片尺寸**自動計算施工加給。

#### 安裝高度分級

適用於需現場安裝的案件：

- **一般（≤200cm）**：0% 加給，站地面即可施工
- **中高（200-300cm）**：+25% 加給，需 A 字梯
- **高空（>300cm）**：+60% 加給，需鷹架

**自動偵測（v0.3.2+）**：

- 在「整面 ÷ 分片」模式下，系統會自動根據整面高度判定安裝高度等級
- 顯示為藍色唯讀框，標註「自動偵測」
- 在「每片 × 數量」模式下，可手動選擇安裝高度

#### 板片尺寸分級

根據板片長邊自動判定：

- **標準（≤180cm）**：0% 加給，一人可搬運
- **大型（180-240cm）**：+20% 加給，需兩人搬運
- **超大型（>240cm）**：+40% 加給，需三人以上搬運

#### 加給計算規則

- 兩個維度的百分比**相加**（非相乘）
- 範例：中高（+25%）+ 大型（+20%）= 總加給 +45%
- 僅影響工資，不影響面料成本
- 工廠製作不安裝的案件，安裝高度選「不安裝」即可

### 系統自動計算

計算器會即時顯示：

- **才數**：根據輸入模式與進位模式計算
- **工資**：基準工資 × (1 + 品質溢價%) × (1 + 施工加給%)
- **面料用量**：根據 fabric-calculator 自動計算裁切方案
- **額外加工**：依勾選項目累計
- **通路倍率後單價**：(工資 + 面料 + 加工) × 通路倍率

### 常見使用情境

#### 情境 1：標準床頭板（一般高度）

- 模式：每片 × 數量
- 尺寸：180×120cm × 2 片
- 安裝高度：一般（≤200cm）→ 0% 加給
- 板片尺寸：標準（≤180cm）→ 0% 加給
- 總加給：0%

#### 情境 2：電視牆（整面分片，中高安裝）

- 模式：整面 ÷ 分片
- 尺寸：360×240cm，橫切 3 片
- 安裝高度：**自動偵測為中高**（240cm）→ +25% 加給
- 板片尺寸：**自動偵測為大型**（360cm 長邊）→ +20% 加給
- 總加給：+45%
- 才數進位：選擇「整面進位」節省成本

#### 情境 3：挑高牆面（高空施工）

- 模式：整面 ÷ 分片
- 尺寸：400×350cm，橫切 4 片
- 安裝高度：**自動偵測為高空**（350cm）→ +60% 加給
- 板片尺寸：**自動偵測為超大型**（400cm 長邊）→ +40% 加給
- 總加給：+100%（工資翻倍）

## 7. 資料到底存在哪裡

這一段是最重要的。

### 正式資料

正式資料存在 Google Sheets，不存在瀏覽器裡。

目前系統會用到的主要工作表包含：

- `客戶資料庫`
- `案件`
- `報價`
- `報價版本`
- `報價版本明細`（v0.3.2 擴展至 32 欄：A-AF）
- `材質資料庫`
- `工資表`
- `系統設定`
- `報價紀錄`
- `報價明細`
- `報價變更紀錄`
- `佣金結算`

### 瀏覽器本機暫存

有些資料只存在你目前使用的這台電腦、這個瀏覽器。

包含：

- 客戶清單快取：`localStorage`，約 5 分鐘
- 材質清單快取：`localStorage`，約 5 分鐘
- 未儲存報價草稿：`localStorage`，自動暫存，超過 24 小時會失效
- 材質收藏 / 最近使用：`localStorage`
- 側邊欄收合狀態：`localStorage`
- 報價欄寬設定：`localStorage`
- 頁面跳轉時暫存要開啟的報價或版本：`sessionStorage`

### 代表什麼

- 換一台電腦，不會看到你這台電腦的草稿與 UI 偏好。
- 清除瀏覽器資料後，草稿、收藏、最近使用會消失。
- 但 Google Sheets 裡的正式資料不會因為清除瀏覽器而消失。

## 8. 常見狀況

### 發現未儲存草稿

系統會自動暫存目前編輯內容。重新開頁面時，如果草稿還在有效時間內，會詢問是否還原。

### 為什麼同事看不到我的草稿

因為草稿是存在你自己的瀏覽器 `localStorage`，不是存在 Google Sheets。

### 為什麼有些單在案件管理看得到，有些在報價紀錄

因為系統目前同時支援新版與舊版資料流：

- 新版：`案件 / 報價 / 報價版本 / 報價版本明細`
- 舊版：`報價紀錄 / 報價明細`

### PDF 公司資料不對

請先到 `系統設定` 更新公司資訊，再重新產生 PDF。

### 材質或客戶剛改完，別台電腦沒看到

先確認資料已經成功儲存到 Google Sheets，再請對方重新整理頁面。
本機快取大約 5 分鐘內可能仍顯示舊資料。

### 整面分片模式下，安裝高度無法手動選擇

這是正常的！在「整面 ÷ 分片」模式下，系統會根據整面高度自動判定安裝高度等級，確保加給計算與實際安裝高度一致。如需手動調整，請切換回「每片 × 數量」模式。

### 兩種才數進位模式差很多怎麼辦

這代表你的分片數量較多或單片尺寸不規則。建議：

- 與客戶溝通時，說明兩種計價方式的差異
- 優先使用「整面進位」模式報價（對客戶較有利，成交率高）
- 橘色提示框會顯示詳細計算過程，方便向客戶說明

## 9. Google Sheets 管理者維護說明

這一段給管理者看，不是一般報價同事日常必讀。

### 管理者平常負責什麼

- 維護 Google Sheets 試算表權限
- 確認系統連到正確的試算表
- 做初始化、備份、必要時遷移舊資料
- 避免同事直接把工作表結構改壞
- 執行版本升級遷移腳本

### 啟用前準備

管理者至少要確認這 4 件事：

1. 已建立或確認要使用的 Google Sheets 試算表。
2. 環境變數已設定：
   - `GOOGLE_SHEETS_SPREADSHEET_ID`
   - `GOOGLE_SERVICE_ACCOUNT_KEY`
3. 試算表已分享給 service account 的 `client_email`，權限至少要 `編輯者`。
4. 第一次使用新試算表時，呼叫一次：
   - `POST /api/sheets/init`

本機常用呼叫方式：

```bash
curl -X POST http://localhost:3001/api/sheets/init
```

### 版本升級遷移（v0.3.1 → v0.3.2）

升級到 v0.3.2 後，需依序執行以下遷移腳本：

```bash
# v0.3.1 遷移（如果尚未執行）
curl -X POST http://localhost:3001/api/sheets/migrate-v03

# v0.3.2 遷移（新增 6 個欄位：AA-AF）
curl -X POST http://localhost:3001/api/sheets/migrate-v032
```

**遷移內容**：

- v0.3.1：新增「安裝高度等級」「板片尺寸等級」「施工加給%」3 欄（W-Y）
- v0.3.2：新增「輸入模式」「整面寬度cm」「整面高度cm」「分片方向」「分片數」「才數進位模式」6 欄（AA-AF）

**安全性**：

- 遷移腳本為冪等操作，可安全重複執行
- 新欄位為 optional，不影響舊資料讀取
- 建議在非營業時間執行遷移

### 目前工作表與維護原則

#### 可以透過系統正常維護的表

- `客戶資料庫`
- `材質資料庫`
- `系統設定`

原則：

- 優先在系統畫面操作，不要直接進 Sheets 改。
- 如果真的要手改，只改資料值，不要改欄位結構。

#### 系統自動維護的表

- `案件`
- `報價`
- `報價版本`
- `報價版本明細`（**v0.3.2 擴展至 32 欄**）
- `報價紀錄`
- `報價明細`
- `報價變更紀錄`
- `佣金結算`

原則：

- 這些表有關聯 ID 與固定欄位順序，平常不要手改。
- 若要緊急修正，也只改單格內容，不要移動列欄或刪整列。

#### 需要特別注意的表

- `工資表`

注意：

- 目前系統初始化時會建立這張表，也會填預設資料。
- 但現行報價邏輯主要仍以程式碼中的常數為準。
- 也就是說，**只改 Google Sheets 裡的 `工資表`，不一定會直接改變報價結果**。
- 如果要正式調整工資規則，應同步檢查程式碼設定。

### 絕對不要做的事

- 不要改工作表名稱。
- 不要改第 1 列標題列。
- 不要插入、刪除、重排欄位。
- 不要手動改系統產生的 ID，例如 `CA-...`、`...-Q01`、`...-V01`、`...-I001`。
- 不要直接刪除 `報價版本` 或 `報價版本明細` 的資料列。
- 不要在正式資料上隨便重跑 `/api/sheets/migrate`。
- 不要在正式資料上隨便重跑 `/api/sheets/init`。

原因：

- 這套系統讀寫 Google Sheets 時，很多欄位位置與工作表名稱是寫死的。
- 一旦工作表結構被改動，畫面可能會顯示空白、資料錯位，或直接寫入失敗。

### `init` 與 `migrate` 路由差在哪裡

#### `POST /api/sheets/init`

用途：

- 建立缺少的工作表
- 重寫每張表的第 1 列標題
- 重設 `系統設定` 與 `工資表` 的預設列

風險：

- 這不是「單純檢查」。
- 重新執行時，會覆寫 `系統設定` 與 `工資表` 目前內容。

適合何時用：

- 第一次接新試算表
- 結構壞掉，需要重建標題列

#### `POST /api/sheets/migrate`

用途：

- 把舊版 `報價紀錄 / 報價明細` 遷移到新版 `案件 / 報價 / 報價版本 / 報價版本明細`

本機常用呼叫方式：

```bash
curl -X POST http://localhost:3001/api/sheets/migrate
```

風險：

- 目前遷移是追加寫入，不會自動判斷你之前有沒有遷移過。
- 如果對同一批舊資料重跑，會產生重複的新資料。

適合何時用：

- 舊系統升級到新版資料模型時
- 僅建議做一次，做之前先完整備份

#### `POST /api/sheets/migrate-v03`（v0.3.1）

用途：

- 新增「安裝高度等級」「板片尺寸等級」「施工加給%」3 欄

#### `POST /api/sheets/migrate-v032`（v0.3.2）

用途：

- 新增「輸入模式」「整面寬度cm」「整面高度cm」「分片方向」「分片數」「才數進位模式」6 欄

### 備份與復原建議

在做下面這些事之前，先備份整份試算表：

- 重跑 `init`
- 跑 `migrate`
- 跑 `migrate-v03` 或 `migrate-v032`
- 大量手動修正資料
- 調整欄位或工作表結構

建議做法：

1. 先在 Google Sheets 複製整份試算表。
2. 用日期命名，例如：`繃布報價_backup_2026-03-22`。
3. 需要回復時，可以：
   - 手動把資料複製回正式表
   - 或直接把 `GOOGLE_SHEETS_SPREADSHEET_ID` 指到備份試算表

### 手動修資料時的安全原則

- 能用系統畫面改，就不要直接改 Google Sheets。
- 如果只是在修公司名稱、電話、備註，改格子內容通常是安全的。
- 如果是案件、報價、版本關聯問題，先備份再改。
- `啟用` 欄請明確填 `TRUE` 或 `FALSE`，不要留空。

特別注意：

- `材質資料庫` 的 `啟用` 若不是 `TRUE`，資料可能不會顯示在系統中。
- 客戶或材質手動改完後，前端快取可能 5 分鐘內還是舊資料，先重新整理再確認。
- `報價版本明細` 現在有 32 欄（A-AF），手動編輯時注意不要改錯欄位

### 常見故障排查

#### 畫面顯示 `Google Sheets 未設定`

先檢查：

- `GOOGLE_SHEETS_SPREADSHEET_ID` 是否存在
- `GOOGLE_SERVICE_ACCOUNT_KEY` 是否是合法 JSON
- 目前啟動的環境是否真的有讀到 `.env.local`

#### 系統打得開，但客戶 / 材質 / 設定是空的

先檢查：

- 試算表是否有分享給 service account
- 工作表名稱是否完全正確
- 標題列是否被改掉
- 是否連到了錯的 spreadsheet ID

#### 改了 `工資表`，報價卻沒變

這是目前系統設計上的已知狀況。

- 現行計價邏輯主要還是讀程式碼常數。
- 所以單改 Google Sheets 的 `工資表`，不保證會影響前端計算結果。

#### 重整後資料怪怪的

先檢查最近是否有人做過：

- 手動改欄位順序
- 重跑 `init`
- 重跑 `migrate`
- 直接刪除版本列或明細列

#### v0.3.2 升級後，舊報價無法正常顯示

先確認：

- 是否已執行 `migrate-v032` 遷移腳本
- 檢查 `報價版本明細` 工作表是否已擴展到 32 欄（A-AF）
- 舊資料的新欄位應該為空，這是正常的（向下相容）
